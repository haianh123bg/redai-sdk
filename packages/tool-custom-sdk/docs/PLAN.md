# Kế hoạch tài liệu kiến trúc

## Mục tiêu
- Có tài liệu kiến trúc tổng quan cho SDK (import -> repo -> tool gen -> execute).
- Liên kết rõ ràng từ bộ docs hiện có.
- Cập nhật sitemap để bao gồm trang kiến trúc.

## Các bước
1) Khảo sát codebase, xác định luồng chính và entrypoints.
2) Thiết kế cấu trúc ARCHITECTURE.md (tổng quan, thành phần, luồng dữ liệu, policy/auth, giới hạn).
3) Tạo docs/ARCHITECTURE.md.
4) Cập nhật docs/README.md và docs/SUMMARY.md để liên kết tài liệu kiến trúc.
5) Cập nhật docs/sitemap.json và docs/sitemap.xml.
6) Rà soát docs, bổ sung mục "Tình trạng hiện tại" và "Roadmap ngắn" nếu cần.
